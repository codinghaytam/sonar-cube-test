## 1. Le Pattern Singleton
Le pattern Singleton est un pattern de création d'objet dont l'objectif est de fournir une seule instance pour une classe spécifique.

Les classes de configuration d'une application ou de connexion à une base de données nécessitent souvent la présence d'une instance unique. L'utilisation de plusieurs instances de ces classes peut produire des incohérences de données et causer des problèmes de fonctionnement de l'application.

Le pattern Singleton est un pattern de creation d'objet, son objectif nécessite a fournir une seul instance pour une classe specifique.

## 2. Le Pattern Holder 
Le Pattern Holder est un pattern qui permet d'obtenir un code performant et thread-safe sans nécessiter une synchronisation manuelle.
